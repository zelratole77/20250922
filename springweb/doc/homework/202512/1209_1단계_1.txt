# 마무리 문제

1.  [개념] Spring에서 GET 방식과 POST 방식을 하나의 메서드 혹은 각각의 메서드로 모두 처리하고자 할 때 사용되는 기본 어노테이션과 코드 형식을 정리해 보세요.

2.  [실습] 다음 시나리오를 충족하는 Controller 코드를 작성해 보세요.

  - 요청 URL: /login
  - 상황: 사용자가 로그인 페이지에 처음 접속할 때(GET)는 아이디/비밀번호 입력 폼으로 이동하고, 폼을 입력 후 제출버튼을 눌렀을 때(POST)는 실제 로그인 처리 로직이 실행되도록 구성하려고 합니다.
  - 조건: @RequestMapping을 사용하여 GET과 POST를 구분하는 설정을 각각 작성하세요.

3.  [개념] Spring에서 요청 파라미터(Request Parameter)가 넘어오지 않을 경우 발생하는 예외를 방지하기 위해, @RequestParam 어노테이션을 사용하여 필수 여부(required) 해제 및 기본값(defaultValue)을 설정하는 방법을 예제와 함께 설명해 보세요.

4.  [실습] 다음 시나리오를 충족하는 메서드 시그니처(매개변수 부분)를 작성해 보세요.

  - 요청 URL: /cafe/order
  - 상황: 카페 주문 시스템입니다. 사용자가 메뉴(menu)와 수량(count)을 선택합니다.
  - 조건:
    1.  메뉴는 필수 입력값입니다.
    2.  수량은 입력하지 않을 경우 기본적으로 1잔으로 처리되어야 합니다.
    3.  수량 파라미터가 없어도 에러가 발생하면 안 됩니다.

5.  [개념] Spring에서 다수의 요청 매개변수를 개별 변수가 아닌 객체(Command Object/DTO)로 한 번에 받을 때의 장점을 서술하고, 이를 구현하는 예제 코드를 기술해 보세요.

6.  [실습] 다음 시나리오를 위해 필요한 VO(Value Object) 클래스와 Controller 메서드를 구상해 보세요.

  - 상황: 회원가입 페이지에서 사용자가 아이디, 비밀번호, 이름, 전화번호, 이메일, 주소, 나이 등 7개의 정보를 입력하여 전송합니다.
  - 문제: 위 7개의 데이터를 각각 @RequestParam으로 받지 않고, MemberVO라는 객체를 통해 한 번에 받는 코드를 작성하세요. (MemberVO 클래스의 멤버 변수 선언 포함)

7.  [개념] Controller에서 문자열이나 정수 같은 단일 데이터를 Model 객체에 담아 View(JSP 등)로 전달하는 기본 코드 형식을 예제를 통해 기술해 보세요.

8.  [실습] 다음 시나리오를 구현해 보세요.

  - 상황: 오늘의 행운 점수 게임입니다.
  - 조건:
    1.  Controller에서 1부터 100 사이의 랜덤한 정수를 생성합니다.
    2.  생성된 숫자를 luckyScore라는 이름으로 Model에 담습니다.
    3.  View 페이지로 이동하여 해당 점수를 출력할 수 있도록 준비합니다.



9.  [개념] Spring Controller에서 배열(Array)이나 리스트(List) 형태의 데이터를 Model에 저장하고, View에서 반복문(JSTL forEach 등)을 통해 출력하는 기본 코드를 설명해 보세요.

10. [실습] 다음 시나리오를 구현해 보세요.

  - 상황: 우리 반의 급식 당번 3명의 명단을 화면에 출력하려고 합니다.
  - 조건:
    1.  Controller에서 "철수", "영희", "민수" 세 명의 이름을 가진 List\<String\> 혹은 배열을 생성합니다.
    2.  이 리스트를 Model에 담아 View로 전송합니다.
    3.  (머릿속으로 상상해보기) View에서는 이 명단을 어떻게 반복해서 출력해야 할까요?



11. [개념] Spring에서 데이터를 단순 변수가 아닌 객체(VO/DTO) 형태로 Model에 설정하여 보낼 때, Controller에서의 코드와 View(EL 표기법)에서 객체의 속성을 호출하는 형식을 설명해 보세요.

12. [실습] 다음 시나리오를 구현해 보세요.

  - 상황: 쇼핑몰의 상품 상세 페이지입니다.
  - 조건:
    1.  Product라는 클래스(속성: name, price)가 있다고 가정합니다.
    2.  Controller에서 name은 "게이밍 마우스", price는 50000인 Product 객체를 생성합니다.
    3.  이 객체를 productInfo라는 키값으로 Model에 담습니다.
    4.  View에서 상품의 이름과 가격을 각각 출력하려면 어떤 EL 표기법(${...})을 사용해야 하는지 작성해 보세요.

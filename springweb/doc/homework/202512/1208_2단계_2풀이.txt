### 📂 테마 1: "망해가는 '스프링 버거' 키오스크 살리기" (1번~10번)
> **시나리오:** 당신은 막 개업했지만 주문 시스템 오류로 손님들이 화를 내고 있는 '스프링 버거'의 긴급 투입된 개발자입니다. `application.properties` 설정부터 꼬여있고, 주문이 주방으로 전달되지 않습니다.

**1. [설정] 간판 달기 (ViewResolver)**
* **문제:** `application.properties`가 비어있습니다. `/WEB-INF/views/burger/` 폴더 안에 있는 `.jsp` 파일들이 컨트롤러에서 리턴값만으로(예: return "menu") 호출될 수 있도록 접두사(prefix)와 접미사(suffix)를 설정하세요.
* **Rubric:**
    * 상: prefix, suffix가 정확하여 404 에러 없이 뷰가 로딩됨.
    * 하: 경로 설정 오류로 404 에러 발생.

**2. [충돌] 쌍둥이 알바생 구분 (Bean Name)**
* **문제:** `BurgerController.java`와 `OrderController.java`가 있는데, 실수로 개발자가 두 클래스 모두에 `@Controller("master")`라는 별칭을 붙여 서버가 켜지지 않습니다. `OrderController`의 빈 이름을 "subMaster"로 변경하여 충돌을 해결하세요.
* **Rubric:**
    * 상: 서버 구동 시 `BeanDefinitionParsingException` 없이 정상 실행됨.
    * 하: 중복된 빈 이름 오류로 서버 시작 실패.

**3. [라우팅] 메뉴판 보여주기 (Simple Get)**
* **문제:** 사용자가 `http://localhost:5050/menu`로 접속했을 때, `burger/menuList.jsp`가 보이도록 컨트롤러 메서드를 작성하세요.
* **Rubric:**
    * 상: URL 호출 시 메뉴판 jsp가 정확히 렌더링됨.
    * 하: 404 에러 혹은 빈 화면 출력.

**4. [데이터] 햄버거 상세 조회 (QueryString)**
* **문제:** 손님이 특정 버거를 클릭했습니다. `http://localhost:5050/detail?burgerName=치즈버거` 형태로 요청이 옵니다. 컨트롤러에서 이 이름을 받아 콘솔에 "주문 메뉴: 치즈버거"라고 출력하고 상세 페이지로 이동하세요.
* **Rubric:**
    * 상: 쿼리 스트링 값이 콘솔에 정확히 출력됨.
    * 하: 파라미터 매핑 실패(400 Bad Request) 또는 null 출력.

**5. [형변환] 세트 메뉴 가격 계산 (int Param)**
* **문제:** `calculate?price=5000&count=2` 요청이 들어옵니다. 두 값을 `int`형으로 받아 총액(10,000)을 계산하여 콘솔에 찍고 결제 페이지로 넘기세요.
* **Rubric:**
    * 상: String이 아닌 int로 매개변수를 선언하여 자동 형변환 및 연산 성공.
    * 하: String으로 받아 파싱하거나, 500 에러(타입 불일치) 발생.

**6. [보안] 결제 요청은 은밀하게 (POST)**
* **문제:** 결제 버튼을 누르면 `/pay` URL로 요청이 갑니다. 카드 번호가 주소창에 노출되면 안 되므로 `<form method="post">`로 전송된 요청만 처리하도록 메서드를 작성하세요. (GET으로 접근 시 에러 페이지나 거부 메시지 출력)
* **Rubric:**
    * 상: POST 요청은 정상 처리, 주소창 입력(GET) 시 405 Method Not Allowed 확인.
    * 하: GET/POST 구분 없이 모두 처리됨.

**7. [중복] "주문 확인" vs "주문 확정" (Same URL)**
* **문제:** URL은 `/orderCheck`로 동일합니다. 단순히 페이지를 볼 때(GET)는 "주문 확인창"을 보여주고, 확인 버튼을 눌러 데이터를 보낼 때(POST)는 "주문 완료" 처리를 하도록 메서드 오버로딩(URL 매핑 분리)을 구현하세요.
* **Rubric:**
    * 상: 동일 URL에 대해 HTTP 메서드 별로 다른 메서드가 실행됨.
    * 하: `IllegalStateException`(Ambiguous mapping) 발생.

**8. [다중] 토핑 추가 폭탄 (Multi Param)**
* **문제:** 햄버거에 토핑을 3개 추가합니다. `custom?t1=치즈&t2=베이컨&t3=토마토`. 각각 변수로 받아서 "토핑: 치즈, 베이컨, 토마토" 형식으로 콘솔에 출력하세요.
* **Rubric:**
    * 상: 3개의 파라미터가 모두 누락 없이 매핑됨.
    * 하: 파라미터 이름 불일치로 일부 값 누락(null).

**9. [경로] 정적 리소스 (Static)**
* **문제:** 햄버거 이미지가 엑박(X)으로 뜹니다. `src/main/resources/static/img/burger.png`에 파일을 넣고, JSP에서 이 이미지를 불러오는 태그를 작성하세요. (DispatcherServlet을 거치지 않는 경로 이해)
* **Rubric:**
    * 상: 브라우저에서 이미지가 정상적으로 로딩됨.
    * 하: 404 에러 (경로 설정 미숙).

**10. [전체] 키오스크 마감 (Controller -> View Data)**
* **문제:** 관리자 페이지입니다. `/admin/close`로 요청하면 컨트롤러에서 `Model`을 사용해 `msg="금일 영업 종료"`를 담아 `close.jsp`로 보냅니다. JSP에서는 `${msg}`를 alert창으로 띄우세요.
* **Rubric:**
    * 상: 컨트롤러의 데이터가 뷰(JSP/JS)까지 전달되어 실행됨.
    * 하: 데이터 전달 실패 또는 JSP EL 표기법 오류.

### 📂 테마 2: "테란 사령부(Command Center) 시스템 구축" (11번~20번)
> **시나리오:** 당신은 테란의 기술 장교입니다. SCV와 마린을 생산하고 자원을 관리하는 사령부의 메인 시스템을 Spring MVC로 구축해야 합니다.

**11. [매핑] SCV 생산 명령**
* **문제:** `/terran/build?unit=SCV` 요청 시 컨트롤러에서 유닛 이름을 받아 "SCV 생산 시작"을 로그로 남기고 `factory.jsp`로 이동하세요. 단, `unit` 파라미터 이름이 `target`으로 들어오더라도 처리되도록 `@RequestParam`의 옵션을 활용해 매핑하세요.
* **Rubric:**
    * 상: 요청 파라미터명과 메서드 변수명이 달라도 매핑 성공(`@RequestParam("target")`).
    * 하: 400 Bad Request 에러 발생.

**12. [로직] 미네랄 채취량 계산**
* **문제:** `/gather?scvCount=5&mineralPerScv=8` 요청이 옵니다. 컨트롤러 내부에서 총 자원(5 * 8)을 계산하여 콘솔에 출력하고 뷰를 호출하세요.
* **Rubric:**
    * 상: 파라미터 형변환 및 비즈니스 로직(곱셈) 정상 수행.
    * 하: 연산 오류 또는 문자열 연결("58") 발생.

**13. [조건] 서플라이 디포 부족**
* **문제:** `/train/marine?current=10&max=10` 요청이 옵니다. 컨트롤러에서 현재 인구수(current)가 최대 인구수(max)와 같으면 `error_supply.jsp`로, 여유가 있으면 `success.jsp`로 리턴하는 분기 처리를 작성하세요.
* **Rubric:**
    * 상: 조건에 따라 서로 다른 뷰 파일이 호출됨.
    * 하: 조건문 오류로 항상 같은 뷰만 호출됨.

**14. [배열] 부대 지정 (Array/List Param)**
* **문제:** 여러 유닛을 한 번에 선택합니다. `select?units=marine&units=medic&units=tank`. 컨트롤러에서 `List<String>` 또는 `String[]`으로 받아 향상된 for문으로 모든 유닛 이름을 출력하세요.
* **Rubric:**
    * 상: 다중 값을 리스트/배열로 정확히 바인딩하여 순회 출력.
    * 하: 첫 번째 값만 받아오거나 타입 에러 발생.

**15. [객체] 시즈모드 변환 (Boolean)**
* **문제:** `/tank/mode?siege=true` 요청을 받습니다. boolean 타입으로 받아 `true`면 "시즈모드 변신", `false`면 "퉁퉁퉁"을 출력하세요.
* **Rubric:**
    * 상: 문자열 "true"가 boolean 타입으로 자동 변환되어 로직 수행.
    * 하: 문자열 비교(String equals) 사용 등 비효율적 처리.

**16. [뷰단] 자원 현황판 (Model)**
* **문제:** 컨트롤러에서 미네랄 500, 가스 200 정보를 `Model` 객체에 담습니다 (`addAttribute`). 뷰(JSP)에서는 이 정보를 받아 "자원: M(500) / G(200)" 형태로 출력하세요.
* **Rubric:**
    * 상: Model 객체 활용 및 JSP EL 표기법(`${key}`) 숙지.
    * 하: 데이터가 뷰에 나타나지 않음.

**17. [POST] 전술 핵 발사 승인**
* **문제:** 핵 발사는 중요하므로 반드시 POST 방식이어야 하며, 암호가 필요합니다. `/nuclear/launch`로 `password=1234`를 보냅니다. 암호가 맞으면 `launch.jsp`, 틀리면 `denied.jsp`로 이동하세요.
* **Rubric:**
    * 상: POST 매핑 및 파라미터 검증 로직 구현.
    * 하: GET으로 접근 가능하거나 암호 검증 누락.

**18. [경로] 정찰기 경로 단축**
* **문제:** 뷰 폴더 구조가 `/WEB-INF/views/terran/unit/air/wraith.jsp`로 깊습니다. `application.properties` 설정을 건드리지 않고, 컨트롤러 리턴값만으로 이 파일을 정확히 찾아가게 하세요. (기본 prefix가 `/WEB-INF/views/`라고 가정)
* **Rubric:**
    * 상: 리턴 문자열에 하위 폴더 경로를 정확히 포함 (`terran/unit/air/wraith`).
    * 하: 404 에러.

**19. [예외] 건물 띄우기 (Default Value)**
* **문제:** `/liftOff` 요청 시 높이(`height`) 파라미터가 오지 않을 수도 있습니다. 파라미터가 없으면 기본값(defaultValue)으로 0을 설정하여 에러를 방지하세요.
* **Rubric:**
    * 상: 파라미터 누락 시 400 에러 대신 기본값 0이 적용됨.
    * 하: 파라미터 없을 시 400 Bad Request 발생.

**20. [종합] 배틀크루저 야마토 포**
* **문제:** `/battlecruiser/yamato` (POST). 에너지(`energy`)가 150 이상일 때만 발사 성공. 성공 시 `Model`에 "발사 성공" 메시지를 담아 뷰로 보내고, 에너지 부족 시 "마나 부족"을 담아 보내세요.
* **Rubric:**
    * 상: 파라미터 수신 -> 조건 로직 -> 모델 저장 -> 뷰 리턴의 MVC 흐름 완벽 구현.
    * 하: 로직 흐름이 끊기거나 데이터 전달 실패.

### 📂 테마 3: "K-디지털 트레이닝 학원 관리 시스템" (21번~30번)
> **시나리오:** KDT 과정 강사로서 학생들의 출석, 성적, 상담을 관리하는 LMS(Learning Management System)의 백엔드 프로토타입을 만듭니다.

**21. [검색] 수강생 조회 (DAO Simulation)**
* **문제:** 검색창에 이름(`name`)을 입력하고 엔터(`searchStudent?name=홍길동`). 컨트롤러는 이 이름을 받아 가상의 DB(메서드 내 하드코딩된 Map이나 if문)에서 해당 학생의 나이와 과정을 찾아내어 콘솔에 출력하세요.
* **Rubric:**
    * 상: 입력값에 따라 다른 결과 데이터를 도출하는 로직 구현.
    * 하: 입력값과 무관한 고정된 결과만 출력.

**22. [등록] 신규 과정 개설 (Form to Controller)**
* **문제:** JSP에 `<form action="createCourse" method="post">`를 만들고 과정명, 강의실, 정원 3개 값을 입력받으세요. 컨트롤러는 이를 받아 "과정명: OOO, 강의실: OOO호 개설됨"을 출력하세요.
* **Rubric:**
    * 상: HTML Form 태그와 Controller `@PostMapping` 파라미터의 일치.
    * 하: 파라미터 이름 불일치로 데이터 수신 실패.

**23. [객체] 성적 입력 (Command Object)**
* **문제:** 국어, 영어, 수학 점수가 넘어옵니다. 파라미터를 하나씩(`@RequestParam`) 받지 말고, `Score`라는 VO(Value Object) 클래스를 만들어 한 번에 받으세요. (커맨드 객체 사용)
* **Rubric:**
    * 상: 요청 파라미터 이름과 객체의 필드명이 일치하여 자동 매핑됨.
    * 하: 객체 매핑 실패 (세터 메서드 누락 등).

**24. [리다이렉트] 상담 신청 후 이동**
* **문제:** `/consult/apply` (POST)에서 상담 신청을 처리한 후, 브라우저가 새로고침 시 중복 신청되지 않도록 결과 페이지나 메인 페이지로 `redirect:` 하세요.
* **Rubric:**
    * 상: `return "redirect:..."` 사용 및 URL 변경 확인.
    * 하: `forward`(일반 리턴) 사용하여 새로고침 시 중복 요청 발생 위험.

**25. [Model] 출석부 출력 (List in Model)**
* **문제:** 컨트롤러에서 학생 3명의 이름이 담긴 `List<String>`을 생성하여 `Model`에 담습니다. 뷰(JSP)에서는 JSTL `<c:forEach>` 혹은 스크립트릿을 이용해 목록을 출력하세요.
* **Rubric:**
    * 상: 컬렉션 데이터가 뷰로 전달되어 반복문으로 출력됨.
    * 하: 객체 주소값이 출력되거나 반복문 구현 실패.

**26. [필수] 지각 사유서 (Required=false)**
* **문제:** `/attendance?status=late` 요청입니다. 지각(`late`)인 경우 `reason`(사유) 파라미터가 필수지만, 출석(`present`)인 경우 사유가 필요 없습니다. `reason` 파라미터의 필수 여부(`required`)를 false로 설정하고 null 처리를 하세요.
* **Rubric:**
    * 상: 파라미터가 없어도 에러 없이 null로 처리됨.
    * 하: 파라미터 누락 시 400 에러.

**27. [수정] 개인정보 수정 (DTO)**
* **문제:** 회원 수정 폼에서 `id`, `pw`, `address`, `phone` 4가지 정보를 POST로 보냅니다. 이를 DTO 객체로 받은 후, `pw`가 "1234"일 때만 "수정 완료" 뷰를 보여주세요.
* **Rubric:**
    * 상: DTO 매핑과 단순 비즈니스 로직(비번 검증)의 결합.
    * 하: 로직 누락.

**28. [디버깅] 404의 늪**
* **문제:** 컨트롤러는 `@GetMapping("student/list")`이고, 뷰 리턴값은 `"student/list"`입니다. 실제 파일은 `/WEB-INF/views/student/list.jsp`에 있습니다. `application.properties` 설정이 `prefix=/WEB-INF/views/`, `suffix=.jsp`일 때, 이 코드가 왜 404가 날 수 있는지(혹은 정상 작동하는지) 파일 경로를 분석하여 정상 작동하도록 코드를 확정하세요.
* **Rubric:**
    * 상: 폴더 구조와 리턴값의 관계를 명확히 이해하여 경로 설정.
    * 하: 중복 경로 (`/WEB-INF/views/student/student/list.jsp` 등) 발생으로 404.

**29. [AJAX 대응] 비동기 아이디 중복 체크 (ResponseBody)**
* **문제:** (심화) 화면 이동 없이 데이터만 리턴해야 합니다. `/checkId?id=test` 요청에 대해 뷰 페이지가 아닌 문자열 "EXIST" 또는 "OK"를 그대로 응답 본문에 싣도록 `@ResponseBody`를 사용하여 구현하세요.
* **Rubric:**
    * 상: JSP 파일을 찾지 않고 텍스트 데이터 자체가 브라우저에 출력됨.
    * 하: "EXIST.jsp"를 찾으려는 404 에러 발생.

**30. [최종] 성적표 발송 시스템 (MVC Full Flow)**
* **문제:**
    1.  사용자가 `/score/req` (GET)로 성적 조회 폼 요청.
    2.  폼에서 이름, 학번 입력 후 `/score/result` (POST) 전송.
    3.  컨트롤러에서 학번이 1000번대면 "1강의실", 2000번대면 "2강의실" 정보를 추가로 생성.
    4.  이름, 학번, 강의실 정보를 `Student` 객체에 담아 `Model`로 전달.
    5.  `result.jsp`에서 위 3가지 정보를 깔끔한 테이블로 출력.
    위 전체 흐름을 구현하세요.
* **Rubric:**
    * 상: GET(폼) -> POST(처리) -> Model(데이터) -> View(출력)의 전체 사이클 완성.
    * 하: 단계 중 하나라도 누락되어 흐름이 끊김.

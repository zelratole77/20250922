
### 1. [개념] AJAX를 이용한 아이디 중복 확인 프로세스

회원가입 시 비동기 통신(AJAX)을 활용하여 아이디 중복 여부를 확인하는 전체 흐름에 대한 문제입니다.

1. 사용자가 아이디를 입력하고 중복 확인 버튼을 눌렀을 때, 화면의 전환 없이 서버와 데이터를 주고받기 위해 사용하는 기술은 무엇입니까?
2. Controller에서 문자열 데이터나 JSON 객체를 View(JSP)를 거치지 않고 직접 클라이언트에 응답으로 보낼 때 사용하는 어노테이션은 무엇입니까?
3. MyBatis 환경에서 SQL의 COUNT(*) 결과값(정수)을 Service단으로 넘겨주기 위해 DAO 인터페이스에서 선언해야 하는 리턴 타입은 무엇입니까?
4. 클라이언트(JS)에서 $.ajax를 호출할 때 서버로부터 성공적으로 응답을 받았을 때 실행되는 콜백 함수 속성명은 무엇입니까?

### 1. [실습] 실시간 비밀번호 일치 확인 시스템

아이디 중복 체크와 동일한 원리를 이용하여, 사용자가 '비밀번호'와 '비밀번호 확인' 칸에 값을 입력할 때마다 실시간으로 두 값이 일치하는지 검증하는 기능을 구현해 봅시다.

* 상황: 보안 요원인 당신은 비밀 기지의 출입 패스워드 설정 시스템을 만들고 있습니다. 두 입력창의 값이 다르면 "불일치", 같으면 "확인 완료"라는 메시지를 화면에 즉시 띄워야 합니다.
* 요구사항:
1. JSP에서 password와 passwordConfirm 두 개의 input 태그를 만드세요.
2. 두 input에 값이 입력될 때마다(keyup 이벤트) AJAX를 통해 서버로 두 값을 보냅니다.
3. Controller에서는 두 값을 비교하여 일치 여부를 문자열로 리턴합니다.
4. 결과를 받아 화면의 특정 span 태그에 출력하세요.

1. 사용자가 검색창에 키워드를 입력하여 검색 버튼을 눌렀을 때, Controller의 매거진 매핑 메서드에서 검색 조건(VO/DTO)을 파라미터로 받기 위해 주로 사용하는 방식은 무엇입니까?

3. Service 단에서 검색 키워드가 null인 경우 빈 문자열("")로 치환하거나 앞뒤에 "%"를 붙여주는 이유는 무엇입니까?

### 2. [실습] "오늘 뭐 먹지?" 랜덤 메뉴 추천 서비스

맛집 관리 시스템을 응용하여, DB에 저장된 점심 메뉴 중 특정 카테고리(한식, 중식 등)를 선택하면 해당 카테고리의 메뉴 하나를 랜덤으로 추천해주는 기능을 구현해 봅시다.

* 상황: 매일 점심 결정을 못 하는 동료들을 위해 '랜덤 점심 추천기'를 제작합니다.
* 요구사항:
1. 화면에 '한식', '일식', '양식' 버튼을 만듭니다.
2. 버튼을 클릭하면 해당 카테고리명이 서버로 전송됩니다.
3. DAO/Service에서는 해당 카테고리의 리스트를 가져온 뒤, Java의 Math.random()을 활용해 그중 한 개의 메뉴(LunchMenu 객체)만 반환합니다.
4. AJAX 성공(success) 시, 반환된 메뉴의 이름과 가격을 화면에 큰 글씨로 출력하세요.

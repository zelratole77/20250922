# 스프링 파일 만들시 주의 사항
1. 스프링은 동일한 이름으로 객체를 컨테이너 메모리에 올리지 못한다.
	class 이름이 객체 이름으로 만들어지기에 가능한한 @로 설정한 클래스는 다른 이름으로
	만들다(다른 패키지라도) ==> 단, @Controller("name01")로 바꾸어 주면
	객체 이름이 바꾸어져 class명도 동일하게 사용하능하다.
2. url mapping이름도 동일한 이름으로 사용하지 못  한다.
	http://localhost:5050/start로 호출하는데 어떤 클래스의 어떤 메서드를 호출할지
	몰라서 발생한다. 혹시, 이런 에러가 나면 메서드 url이름을 다르게 만든다.
	단,(여기서도 예외)
	같은 이름이라도, @GetMapping("start")이 있으면
	@PostMapping("start")는 가능하다 왜 그럴까?? get방식과 post방식이 다르기 때문이다.

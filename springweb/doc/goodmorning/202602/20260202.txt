## 제 2부: 주관식 (기술형 5문항)

**1. Flask 프레임워크를 'Micro Web Framework'라고 부르는 이유를 Django와 비교하여 서술하시오.** 
	(본문에 언급된 비유를 활용하여 설명할 것)
	Django : 호화 캠핑장처럼 웹 개발에 필요한 모든 기능(DB, 인증, 관리자페이지 등)이 기본적으로 포함된
	fullstack 프레임워크인 반면, Flask는 배낭처럼 개발에 필요한 최소한의 뼈대(Core)만 제공합니다.
	개발자가 필요한 기능(데이터베이스, 로그인 등)을 라이브러리 형태로 직접 선택하여 추가/조립할 수 있어 가볍고 유연하기 때문에
	Micro라고 부릅니다.

**2. 제공된 `updateProc` 함수 내에서 사용자가 특정 항목(예: 가격)을 변경하지 않고 Enter 키를 입력했을 때, 
	데이터가 `0`이나 `null`이 되지 않고 기존 가격이 유지되는 원리(코드 로직)를 설명하시오.**
	input() 함수에서 아무것도 입력하지 않고, enter를 치면 빈 문자열이 반환됩니다.
	python에서 빈 문자열은 논리적으로 False로 간주됩니다.
	if price : 를 통해서 입력되었을 때, 처리하게끔 효율적으로 처리하면 객체가 들어있는 기존의 가격 정보가 
	덮어씌워지지 않고 그대로 보존 되는 처리를 할 수 있다.

**3. Python DB 연동 코드 중 `dataProc(delSql, (delItem_id,), None, "DELETE")`에서 `(delItem_id,)` 
	부분에 쉼표(,)가 반드시 들어가야 하는 문법적 이유는 무엇인가?**
	DELETE FROM 테이블 WHERE 키 = :1
	형식으로 매개변수로 튜플데이터가 필요로 할때, 입력된 값에서 ,를 붙여 튜플형입을 선언하게 한다.

**4. Flask 애플리케이션 코드에서 `@app.route('/hello')` 바로 아래에 정의된 함수(`def hello():`)의 역할과, 
	이 함수가 반환(return)하는 값은 웹 브라우저에서 어떻게 처리되는지 설명하시오.**
	URL과 VIEW단을 호출하게 처리할 때, @app.route('/hello')는 URL 관련 정보 return 뷰단 호출 정보를 처리한다.

**5. CLI(Command Line Interface) 기반의 데이터 관리 프로그램에서 '상세 조회' 기능을 수행한 후, 
	해당 화면에서 '수정', '삭제' 기능으로 바로 연결되도록 메뉴를 구성하는 것이 사용자 경험(UX) 측면이나 프로세스 흐름상 
		어떤 이점이 있는지 서술하시오.**
	CLI(Command Line Interface) : 콘솔창에서 데이터 처리해주는 방식으로 사용자가 데이터를 수정 삭제하려면 그
	내용이 맞는지 확인하는 절차가 필요하다. 상세 조회를 통해 데이터를 확인한 상태를 유지하면서 바로 수정/삭제를 수행하면,
	사용자가 다시 메인 메뉴로 나가서 key를 재입력해야 하는 번거로움을 줄일 수 있습니다. 이는 작업의 연속성을 보장하고,
	엉뚱한 데이터를 건드리는 실수를 방지하여, 동선을 최소화하여 프로그램 사용 편의성을 높여줍니다.

**1. 모듈을 호출(import)하는 방식 중, `import 모듈명`과 `from 모듈명 import *`의 차이점을 
'사용할 때의 코드 작성 방식'과 '이름 충돌 가능성' 관점에서 서술하시오.**
import 모듈명 : 모듈명.포함된구성요소를 호출하는 방식 즉, 접두어로 모듈명. as를 붙이면 alias명 사용
	포함된구성요소가 여러 모듈에 중복된 이름이 있으면 반드시 이 형식으로 처리하여야 한다.
from 모듈명 import * : 모듈에 선언된 모든 구성요소를 현재 모듈에서 선언된 것과 동일하게 특정 모듈명
	이나, alias 없이 사용가능하다.

**4. `@dataclass`를 사용한 DTO 정의 시, 리스트 형태의 필드(속성)에 
`posts: List[PostDTO] = []`라고 선언하면 안 되고, 
반드시 `field(default_factory=list)`를 써야 하는 이유는 무엇인가? (메모리 공유 관점에서 설명)**
반드시 해당 DTO만 할당할려고 할때, 사용되는 형식으로 기본 DTO를 field(default_factory=list)
형식으로 설정하면 매번 해당 객체로 할당하여 처리할 수 있기에 DTO를 지정하여 할당할 수 있다.


**5. 데이터베이스 연동의 핵심 5단계(연결 -> 커서생성 -> 실행 -> 확정/인출 -> 종료) 중, 
'커서(Cursor)'가 필요한 이유와 역할에 대해 설명하시오.**
커서가 DB 연결객체를 통해서 만들어져서 sql문장을 실행하고 그 조회된 결과를 데이터를 담아서 리턴하는 메서드로
출력처리할 수 있다.  Connection 길,  Cursor는 그 길을 오가며 일하는 운반자라고 비유할 수 있다.

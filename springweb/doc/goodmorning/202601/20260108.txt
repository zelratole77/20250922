## II. 주관식 (기술형)

**1. FullCalendar에서 일정을 렌더링할 때 사용하는 기본 데이터 구조는 배열과 객체의 조합입니다. 
	`{title: "수업", start: "2026-01-07"}`와 같은 데이터를 담는 Java 클래스(DTO)를 설계할 때, 
	필드명 설정 시 주의해야 할 점을 기술하세요.**
	fullcalendar api에 있는 필수 속성은 반드시 DB 테이블의 컬럼/DTO에 반드시 지정하여야 한다.

**2. 서버(Spring)에서 DB에 저장된 일정 리스트를 가져와 JSON 형태로 리턴했습니다. 
	하지만 FullCalendar 화면에 아무런 일정이 표시되지 않을 때, 체크해야 할 확인 사항 2가지를 기술하세요. 
	(데이터 형식, 속성명 등 관점에서)**
	start, title 속성에 DB 데이터가 설정이 되어 있는지
	start 날짜 형식이 ISO 표준 형식(YYYYMMDD 또는 YYYYMMDDTHHMMSS)으로 되어 있는지 확인

**3. `calendar_room` 테이블을 설계할 때, '예약 시간의 중복'을 논리적으로 막기 위해 
	SQL 또는 Service 로직에서 비교해야 할 조건(필드)들에 대해 기술하세요.**
	DB 회의실기준으로 시작일과 종료일이 겹치지 않는지 확인을 해서 겹치면 리턴값으로 메시지를 전송해서,
	일정이 중복된다고 전달하여야 한다.(비동기적 처리)
	

**4. FullCalendar의 `events` 속성에서 `url`을 사용하는 방법과 사용자 정의 속성 `urlLink`를 사용하는 
	방법의 기능적 차이점을 설명하세요.**
	url로 설정하면 상세화면이 모달창으로 가기전에 바로 링크된 주소로 이동이 되는 단점이 있다..
	그래서 필요에 따라(상세화면 모달창 있을 때) 사용자 정의 urlLink를 처리하여, 모달창 안에서 해당 항목의 주소를
	클릭시, 로딩되게 한다.

**5. 비동기 통신(Ajax)을 사용하여 일정을 추가(Insert)하는 과정을 순서대로 기술하세요. 
	(화면 입력값 추출부터 DB 반영 및 화면 갱신까지)**
	비어 있는 일정이나 시간을 스크롤 시, 입력할 모달창이 로딩되고, 일정내용, 작성자, 상세내용, urllink를 입력 후,
	등록(ajax 비동기적으로 처리)을 하면, DB에 실시간 반영이 되어, 전체 화면에 등록된 일정 확인할 수 있다.

③ URL이 중복되면 서버 시작 시 경고만 발생하고, 실행 시 랜덤으로 하나의 메서드가 실행된다.
④ 메서드 이름이 다르면 URL 패턴이 동일해도 상관없다.

#### II. 주관식 (기술형)

**1. `application.properties`에 설정된 `prefix`와 `suffix` 설정을 참고하여,
	 컨트롤러가 `return "board/list";`를 반환했을 때 최종적으로 완성되는 JSP 파일의 전체 경로를 기술하시오.**
	/WEB-INF/views/board/list.jsp
**2. 스프링 컨트롤러에서 동일한 URL(`req04`)을 사용함에도 불구하고 에러가 발생하지 않고, 
	GET 요청과 POST 요청을 분리하여 처리할 수 있는 이유(매커니즘)를 서술하시오.**
	method방식을 기준으로도 요청 url mapping 구분하여 처리하기 때문이다.
**3. MVC 패턴의 핵심 프로세스 흐름을 참고하여, 사용자가 폼에 데이터를 입력하고 전송(Submit)했을 때 
	DB 조회 후 결과 페이지가 보이기까지의 과정을 `Controller`, `Service`, `DAO`, `Model`, 
	`View` 키워드를 모두 사용하여 순서대로 간략히 기술하시오.**
	Controller ==> Service ==> Dao ==> Service 
	==> Controller(Model 데이터에 DB 정보 mapping)
	==> View (el태그로 모델데이터 출력)

**4. 다음 메서드에서 `http://localhost:5050/req11_one` URL만 브라우저에 입력했을 때(파라미터 없이),
 400 에러가 발생하는 원인을 `@RequestParam`의 속성과 관련지어 설명하시오.**

```java
@GetMapping("req11_one")
public String req11_one(@RequestParam("prodName") String prodName) { ... }
```
@RequestParam("prodName") String prodName 있으면 필수적으로 요청값을 전달하여야 한다.
http://localhost:5050/req11_one?prodName=사과  ==> 형식으로 처리..

**5. 제공된 학습 자료에서 '정적 리소스(CSS/JS)'와 '동적 뷰(JSP)'의 위치는 각각 어디로 설정되어 있는지 
경로를 쓰시오.**
정적 리소스(CSS/JS) : src/main/resources/static 폴드 하위에 프로그램에 의해 변경되지 않는 내용
동적 뷰(JSP) : src/main/webapp 폴드가 기본이고 설정에 의해서 /WEB-INF/views로 처리가 가능하다.

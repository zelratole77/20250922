### [Part 2] 주관식 서술형 5문제

**1. 파일 업로드 기능을 구현할 때, HTML Form 태그에 `enctype="multipart/form-data"` 속성을 반드시 
	지정해야 하는 이유를 서술하시오.**
기본적인 form 전송 방식(application/x-www-form-urlencoded")은 데이터를 텍스트(문자열) 형태로 인코딩하여 전송하기 때문에,
이미지나 문서 같은 바이너리(이진) 데이터를 전송할 수 없습니다.
multipart/form-data를 설정해야만 http 요청 메시지를 여러 부분(part)으로 나누어, 텍스트 데이터와 바이너리(이진) 파일 데이터를
구분하여 서버로 전송할 수 있기 때문입니다.
	

**2. Spring Service 계층에서 `MultipartFile` 객체의 `transferTo()` 메서드가 수행하는 역할과, 
이 메서드를 호출하지 않았을 때 업로드된 파일이 어떻게 되는지 설명하시오.**
MultipartFile 객체가 메모리나 임시 저장소에 가지고 있고 실제 물리적인 경로(서버의 하드디스크 등)에 저장되지 않는다.

**3. `@Autowired(required=false)` 설정이 필요한 상황은 어떤 경우인지, 제공된 `Car`와 `Engine` 클래스의 
관계를 빗대어 설명하시오.**
스프링 컨테이너에 해당 타입(Engine)의 Beean이 존재하지 않더라도 에러를 발생시키지 않고, null 상태로라도 객체(Car)를 생성하고
싶을 때 사용합니다.


**4. 소스 코드에서 `application.properties`에 파일 저장 경로(`user.upload`)를 별도로 분리하여 작성하고, 
	`@Value` 어노테이션으로 주입받아 사용하는 방식의 장점은 무엇인가?**
파일 저장 경로는 개발 환경과 운영 환경이 다를 수 있는데, 소스 코드(java)에 경로를 하드코드하면 환경이 바뀔 때마다 코드를	
수정하고 다시 컴파일해야 하는데, 설정 파일로 분리하면 소스 수정 없이 설정값만 변경하여 바로 적용할 수 있다.
	

**5. `@Component` 어노테이션과 `@Bean` 어노테이션의 차이점을 '설정 위치'와 '사용 목적'의 관점에서 간단히 서술하시오.**
@Component는 클래스 단위에 붙이면 @Bean은 @Configuration 클래스 내부에 메서드 단위에 붙입니다.

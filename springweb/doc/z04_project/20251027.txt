## 🚀 HTML/CSS & Oracle SQL 미니프로젝트 : 사이트 분석 및 구현

한 달간 배운 HTML/CSS 및 Oracle SQL 기초 지식을 활용하여, 실제 운영 중인 웹사이트의 특정 기능을 분석하고 간단하게 클론(Clone) 구현 및 DB를 설계하는 미니프로젝트입니다.

### 1. 프로젝트 개요

* **주제:** 관심 있는 웹사이트 선정 후, 주요 화면 분석, 클론 구현, 및 DB/SQL 도출
* **목표:**
    1.  HTML/CSS를 사용한 정적 화면 구현 역량 강화
    2.  웹사이트 화면의 데이터를 분석하고 필요한 테이블을 역으로 설계하는 능력 배양
    3.  데이터 조회를 위한 기본 SQL 작성 능력 향상
* **기간:** **2025년 10월 27일(월) ~ 2025년 11월 7일(금) (총 2주)**
* **팀 구성:** 2~3명 1조
* **주요 산출물:** 사이트 분석 PPT (Google Slides 활용)

### 3. 프로젝트 진행 순서 (권장)

1.  **대상 사이트 선정 (1~2일 차)**
    * 팀원 간 협의하여 벤치마킹할 사이트를 선정합니다.
    * **(중요!)** 2주 내에 분석 가능한 **'단순한 기능'**을 가진 사이트를 선택해야 합니다.
    * **(예시)**
        * **좋은 예:** 특정 쇼핑몰의 상품 목록/상세 페이지, 간단한 커뮤니티 게시판(목록/상세/쓰기), 영화 예매 사이트의 영화 목록, 로그인/회원가입 페이지 등
        * **나쁜 예:** 네이버/다음 포털 메인, 유튜브, 인스타그램, 넷플릭스 (기능이 너무 복잡하고 방대함)

2.  **사이트 흐름 확인 및 분석 (2~3일 차)**
    * 선정한 사이트의 주요 사용자 흐름(Flow)을 파악합니다.
        * (예: 메인 페이지 -> 로그인 -> 상품 목록 -> 상품 상세 -> 장바구니)
    * 이 중 구현할 핵심 화면 2~3개를 확정합니다.

3.  **구현 항목 도출 (3~5일 차)**
    * **[Front-End]** 선택한 화면을 HTML/CSS로 어떻게 구성할지 구조(Layout)를 잡습니다.
    * **[Back-End]** 해당 화면에 '표시되는 데이터'가 무엇인지 모두 식별합니다.
        * (예: 게시글 제목, 작성자, 작성일, 조회수, 상품 이미지, 상품명, 가격...)
    * **[Database]** 이 데이터를 저장하기 위해 어떤 **테이블**과 **컬럼**이 필요한지 설계합니다.

### 4. 최종 산출물 (Google Slides로 통합)

발표 자료(Google Slides)에는 아래 5가지 내용이 모두 포함되어야 합니다.

**1. 요구사항 정의서**
* 프로젝트 개요 (목표, 기간, 팀원 및 역할)
* 벤치마킹 사이트 선정 이유
* 분석할 기능 범위 정의

**2. 화면 설계서 (스토리보드)**
* 구현하기로 선정한 2~3개 화면의 와이어프레임 또는 레이아웃 스케치
* 각 화면의 주요 구성 요소 설명

**3. 주요 화면 구현 (HTML/CSS)**
* 작성한 HTML/CSS 핵심 코드 일부 (전체 코드는 별도 첨부)
* 실제 브라우저에서 구현된 화면 스크린샷

**4. DB 테이블 도출 및 ERD/SQL**
* 화면 데이터를 기반으로 도출한 **테이블 명세서** (테이블명, 컬럼명, 데이터 타입 등)
* 테이블 간의 관계를 표현한 **ERD** (Entity-Relationship Diagram)
* 해당 화면의 데이터를 조회/입력하는 데 사용될 **주요 SQL문** (예: `SELECT`, `INSERT`)

**5. 프로젝트 정리 (회고)**
* 프로젝트를 진행하며 어려웠던 점
* 새롭게 배운 점 또는 좋았던 점 (팀원별 한마디)
* (선택) 다음 프로젝트에서 도전하고 싶은 과제

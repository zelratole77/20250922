# 중간 발표
- 공 통 -
1. 기본 양식으로 작성자 표현.
2. 주간 단위 진행(팀별, 개인별로 구체적인 업무 내용 진행 계힉/달성 내용표현, 팀별 코드리뷰/단위테스트)
3. 기본 목차 준수..



























4조
1. 주제 한글로도 내용을 알수 있게 (MRI 시각화 웹 플렛폼)로 표현
2. 사이트 소개 - 사용자별 처리 및 권한 내용 추가 필요(접수/MRI촬영 간호사/의사 진단 병원 시스템 도입)
3. 프로젝트 개발자 관점에서 프로젝트 목표(목적에 학습자 관점이 보임)
4. 각 문서에 작성자 표현(요구사항정의서에 빠짐)
5. 요구사항정의서는 사용자에 대한 내용보다 화면설계서에 항목을 추가하여 사용자별 권한표현(보기, 등록/수정/삭제 등)
6. ERD 조금 선명하게 보일 수 있게 합니다. - 거의 안보이네요.

1조
1. 목차에 이번에 진행할 목차를 전체 내용을 다 포함되어 있고, 나중에 추가할 수 있도록 합니다. (구현에 대한/후기 내용빠짐)
2. 역할분담은 공통적인 사항(모두 요구사항/화면설계서/화면구현 진행하는데..) 보다 업무 내용을 표현합니다.
3. 일정이 프로그램단위가 표현되어 있는데, 이것보다 업무 내용과 함께 표현  ex) 가볼만한곳(DB설계,화면구현)
4. 작성자 표기(모든 문서)
5. 화면설계서 작성 틀안에서 표현(양식 참조)
6. ERD는 논리엔티티(한글로)/물리엔티티 둘다 표현

1. 기본 양식으로 작성자 표현.
2. 주간 단위 진행(팀별, 개인별로 구체적인 업무 내용 진행 계힉/달성 내용표현, 팀별 코드리뷰/단위테스트)



2조
1. 제목 GO-MONTH : (월별 여행지 큐레이션)로 한글로 내용도 표현
2. 참고용 사이트 분석은 아주 좋은 내용인데, 목적은 내용을 구체적으로 어떻게 적용할 것인가? - 다음 번은 참고 사이트 분석은 생략하고 바로 어떤 기능을 진행할 것인지를 표시
3. 요구사항정의서 기본 양식 준수 - 분류항목이면 표기. 이 내용을 기반으로 화면설계/db설계를 하므로 구체적인 항목과 권한도 표기하여 세부적으로 표현하기
	EX) 회원가입은 아이디,패스워드, 이름, 권한 입력하여 등록시, 필수항목(아이디/패스워드/이름)에 대한 체크하여 등록이 되게 한다.
4. 작성자가 홍길동? ^^ - 실작성자 넣어주세요.
5. 화면설계서 내용설명이 글자가 너무 작아 내용이 잘 안보입니다. 조금 크게 표현해주세요.(일부페이지)
6. 화면설계서에 구체적인 이미지 표시합니다.(X ==> 실사용 이미지/텍스트 및 데이터 표현)
7. 화면설계서 양식 통일해주세요.
8. 워크플로우가 메뉴구조도와 차이점이 없습니다. 실제 워크플로우 도형을 사용하여 프로세스 흐름을 확인하도록 합니다.
9. erd는 논리erd(한글)과 물리ERD(영문) 나누어 표현합니다.
